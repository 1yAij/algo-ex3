# 图着色可视化 SPA 设计说明

日期：2026-04-26

## 1. 目标

构建一个单页前端应用，用于可视化图着色问题的回溯求解过程，并支持以下策略组合：

- 标准回溯
- Degree Descending（静态度排序）
- MRV（动态最少剩余值）
- Forward Checking（前向检查）

应用目标不是只展示最终结果，而是以教学为主，完整呈现选点、试色、剪枝、失败、回溯和成功的全过程，同时保持良好的视觉体验和可操作性。

## 2. 交付边界

- 最终交付为单个 `index.html`
- CSS 和 JavaScript 全部内嵌在 `index.html`
- 使用 CDN 引入 `vis-network`
- 不依赖后端
- 所有图生成、求解、状态管理、回放逻辑均在浏览器运行

## 3. 页面结构

整体采用三栏工作台布局。

### 左栏：控制与指标

用于输入问题、生成图、设置颜色数和策略、控制回放。

包含以下模块：

1. 图生成器
- 图类型下拉框：随机图 / 网格图 / 环形图 / 自定义边列表
- 快捷按钮：成功示例、失败示例、随机图
- 自定义边列表文本框，支持形如 `0-1, 1-2, 2-3`

2. 颜色配置
- 颜色数滑块，范围 `3-6`
- 色板预览，实时反映可用颜色数量

3. 策略选择
- `Standard Backtracking`
- `Degree Descending`
- `MRV`
- `Forward Checking`

策略组合规则：

- 未选择启发式时，使用原始节点顺序
- 选择 `Degree Descending` 时，使用静态度降序
- 选择 `MRV` 时，MRV 成为主选点规则
- 若同时选择 `MRV + Degree Descending`，则 `Degree Descending` 仅作为平局时的 tie-breaker
- `Forward Checking` 可与 `Degree Descending` 或 `MRV` 组合

4. 播放控制
- `Play`
- `Pause`
- `Next Step`
- `Step Back`
- 动画速度滑块

5. 实时指标
- Nodes Explored
- Backtracks Triggered
- Current Step
- Current Depth
- Solve Status

### 中栏：图画布

使用 `vis-network` 渲染图。

要求如下：

- 节点位置只在建图后确定一次
- 回放过程中不重新布局，避免图形跳动
- 节点显示清晰标签
- 节点旁显示度数或剩余域时，采用轻量级附加标签，不干扰主图阅读
- 支持 hover 和 click，显示当前节点的度数、邻居、颜色、剩余域等信息

### 右栏：分析面板

用于同步解释当前算法行为，默认同时保留事件日志和候选域信息。

包含以下模块：

1. 当前动作摘要
- 当前动作类型
- 当前节点
- 当前尝试颜色
- 当前深度
- 当前策略说明

2. 事件日志
- 记录最近若干步的动作文本
- 例如：选点、尝试颜色、冲突、前向检查、域清空、回溯、成功

3. 剩余域面板
- 展示所有未着色节点的当前 domain
- 用紧凑色块或 `{1,2,3}` 形式表示

4. 排序 / 度数面板
- 展示节点度数
- 展示 Degree Descending 下的静态顺序
- 展示 MRV 当前选择依据

## 4. 默认体验

默认采用折中方案：

- 首次加载展示固定成功示例
- 提供“一键失败示例”按钮，用于直接展示明显的剪枝失败和回溯过程
- 提供“一键随机图”按钮，用于自由探索

这样既保证首次体验稳定，也保留强教学场景。

## 5. 算法架构

核心设计为“同步求解 + 轨迹回放”。

### 5.1 求解接口

```js
solveGraphColoring(nodes, edges, numColors, strategies) => State[]
```

其中 `strategies` 建议结构：

```js
{
  staticDegree: true,
  mrv: true,
  forwardChecking: true
}
```

### 5.2 核心原则

- 不边求解边动画
- 点击运行后，先一次性同步生成完整 `trace`
- 播放器只消费 `trace`

这样可以稳定支持：

- 前进一步
- 后退一步
- 暂停
- 改变播放速度
- 任意回看历史状态

## 6. 状态模型

每一步都记录为一个 `State` 对象。

建议字段如下：

```js
{
  stepIndex,
  actionType,
  focusedNode,
  candidateColor,
  assignments,
  domains,
  nodeOrder,
  visitedCount,
  backtrackCount,
  message,
  prunedNodes,
  failedNode,
  depth
}
```

字段说明：

- `actionType`：动作类型
- `focusedNode`：当前处理节点
- `candidateColor`：当前尝试颜色
- `assignments`：所有节点当前着色结果
- `domains`：所有未着色节点当前可选颜色集合
- `nodeOrder`：当前展示给用户的排序信息
- `visitedCount`：已探索节点数
- `backtrackCount`：已触发回溯数
- `message`：自然语言说明
- `prunedNodes`：前向检查中被更新 domain 的邻居
- `failedNode`：domain 清空的节点
- `depth`：当前递归深度

### 6.1 动作类型

至少支持：

- `SELECT_NODE`
- `TRY_COLOR`
- `COLORING`
- `REJECT_COLOR`
- `FORWARD_CHECK`
- `FORWARD_CHECK_FAILED`
- `BACKTRACKING`
- `SOLUTION`
- `FAILURE`

记录粒度要求如下：

- 选中一个节点，单独记一帧
- 尝试某个颜色，单独记一帧
- 染色成功，单独记一帧
- 前向检查更新邻居域，单独记一帧
- 某节点域被清空，单独记一帧
- 回溯撤销颜色，单独记一帧

这样日志、域面板和主图动画可以严格共用同一时间轴。

## 7. 策略行为定义

### 7.1 Standard Backtracking

- 按原始节点顺序依次选点
- 逐色尝试
- 遇到冲突则换色
- 颜色全部失败则回溯

### 7.2 Degree Descending

- 求解前计算每个节点度数
- 生成静态度降序顺序
- 整个求解期间保持该顺序
- 同度节点按节点编号稳定排序

### 7.3 MRV

- 每一层根据当前 domain 动态选点
- 优先选择剩余合法颜色数最少的未着色节点
- 若平局且启用了 `Degree Descending`，则使用度数高者优先
- 若仍平局，则按节点编号稳定排序

### 7.4 Forward Checking

- 某节点染色后，立即从其未着色邻居的 domain 中删除该颜色
- 若某个邻居 domain 为空，记录 `FORWARD_CHECK_FAILED`
- 该失败不是程序异常，而是搜索树中的正常剪枝结果

## 8. 可视化映射

### 8.1 主图动作映射

- `SELECT_NODE`
  - 当前节点外圈高亮脉冲
  - 其他节点轻微降饱和

- `TRY_COLOR`
  - 当前节点显示候选色预览环

- `COLORING`
  - 节点填充正式颜色
  - 相邻边和邻居短暂提亮

- `REJECT_COLOR`
  - 当前节点外圈闪橙红，表示该颜色非法

- `FORWARD_CHECK`
  - 从当前节点向受影响邻居显示虚线辐射效果
  - 右栏剩余域同步变化

- `FORWARD_CHECK_FAILED`
  - 域清空节点变红
  - 状态区显示 `Domain Empty -> Backtrack`

- `BACKTRACKING`
  - 当前被撤销节点的颜色淡出恢复空白
  - 显示回退说明

- `SOLUTION`
  - 整图提亮
  - 状态条显示成功

- `FAILURE`
  - 状态条显示无解
  - 保留最终失败轨迹

### 8.2 策略专属表现

Degree Descending：

- 节点旁显示小号度数标签
- 右栏展示静态排序列表
- 当前节点在排序列表中同步高亮

MRV：

- 未着色节点旁显示剩余域
- 被选中节点显示更强高亮
- 右栏说明“当前最小剩余值”依据

Forward Checking：

- 每步显示受影响邻居
- 域缩减在右栏实时更新
- 域清空节点与相关边短暂染红

## 9. 数据与模块分层

虽然最终只有一个 `index.html`，但内部按模块组织：

### 9.1 GraphFactory

职责：

- 生成固定成功示例图
- 生成固定失败示例图
- 生成随机图
- 生成网格图
- 生成环形图
- 解析自定义边列表

### 9.2 ColoringSolver

职责：

- 接收图、颜色数、策略配置
- 运行回溯求解
- 输出完整 `trace`

### 9.3 TracePlayer

职责：

- 管理当前播放索引
- 实现播放、暂停、前进、后退
- 管理速度

### 9.4 GraphRenderer

职责：

- 初始化 `vis-network`
- 固定节点位置
- 根据 `State` 更新节点、边、标签、动画效果

### 9.5 UIController

职责：

- 绑定按钮、滑块、复选框、文本框
- 刷新指标
- 刷新日志
- 刷新剩余域面板和排序面板
- 处理错误提示和状态切换

## 10. 错误处理

### 10.1 输入校验

自定义边列表支持形如：

```txt
0-1, 1-2, 2-3
```

处理规则：

- 自动去空格
- 自动去重边
- 过滤自环
- 非法格式立即报错

错误提示应明确，例如：

- 边格式错误
- 图为空
- 节点解析失败
- 颜色数不合法

### 10.2 求解失败

当颜色数不足导致无解时：

- 视为算法正常结果，不是程序错误
- 最终 `trace` 以 `FAILURE` 状态结束
- UI 提示“当前颜色数下无解”

### 10.3 状态失效

若用户修改以下任一配置：

- 图结构
- 图类型
- 颜色数
- 策略

则旧轨迹立即失效，播放器重置到未运行态，避免显示过期数据。

## 11. 性能边界

面向 `10-30` 个节点的教学与实验展示优化。

默认控制如下：

- 随机图默认 `10-14` 个节点
- 中等密度边数
- 示例图优先保证轨迹可读性

防护策略：

- 若状态数超过阈值，则停止生成轨迹并提示：
  “状态过多，建议增加颜色数、减小图规模，或关闭部分高成本组合”

这样可以防止在高密图或极端无解配置下卡死页面。

## 12. 响应式策略

桌面端优先使用完整三栏布局。

窄屏下：

- 左栏保持在顶部
- 中栏图画布居中
- 右栏分析模块折叠到下方

需要保证：

- 主图仍可读
- 控件仍可操作
- 日志和域面板不会挤坏布局

## 13. 测试与验证要求

至少覆盖以下验证场景：

1. 固定成功示例可以完整回放到成功
2. 固定失败示例可以稳定出现 `Domain Empty -> Backtrack`
3. `MRV + Forward Checking` 组合可以正常生成轨迹
4. `Degree Descending + Forward Checking` 组合可以正常生成轨迹
5. 自定义边列表非法输入能正确报错
6. 用户切换图或策略后，旧轨迹被正确清空
7. 前进、后退、暂停、变速都基于同一条稳定轨迹工作

## 14. 实现结论

最终实现方向为：

- 整体替换当前页面
- 保留单文件交付形式
- 采用三栏工作台布局
- 默认成功示例 + 一键失败示例 + 一键随机图
- 采用“同步求解生成完整轨迹，再由播放器回放”的技术路线
- 使用 `vis-network` 作为图渲染库

该方案在教学表达、视觉表现、交互稳定性和后续扩展性之间取得平衡，适合作为课程实验展示页面。
